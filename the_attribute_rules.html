The attribute rules

There are some rules for attribute particularly

  • Attribute value must have qoutes.
    • Example:
      • Don't do this:
        <p>
        <input type=submit value=log-in>
        </p>
      • Do this instead:
        <p>
        <input type="submit" value="log-in">
        </p>
  • Attribute are separated with spaces.
    • Example:
      • Don't do this:
        <p>
        <input type="submit"value="log-in">
        </p>
      • Do this instead:
        <p> <input type="submit" value="log-in">
        </p>
  • Attribute names need to be lowercase.
    • Example:
      • Don't do this:
        <p> <input type="SUBMIT" value="LOG-IN">
        </p>
      • Do this instead:
        <p> <input type="submit" value="log-in">
        </p>
  • Element types have private attribute.
    • Example:
      • Don't do this:
        <p> <input type="submit,log-in">
        </p>
      • Do this instead:
        <p> <input type="submit" value="log-in">
        </p>
  • Public attributes can go on any elements like: class; id=hello
  • No double qoutes inside of doubles qoutes.
    • Example:
      • Don't do this:
        <p> <input "type="submit" value="log-in"">
        </p>
      • Do this instead:
        <p> <input type="submit" value="log-in">
        </p>
