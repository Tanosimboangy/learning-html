The attribute rules

There are some rules for attribute particularly:

  • Attribute value must have quotes: Use quotes to wrap each attribute value.
    • Example:
      • Don't do this:
        <p>
        <input type=submit value=log-in>
        </p>
      • Do this instead:
        <p>
        <input type="submit" value="log-in">
        </p>
  • Attribute are separated with spaces: If the attribute is more than one, you should separate them by putting spaces.
    • Example:
      • Don't do this:
        <p>
        <input type="submit"value="log-in">
        </p>
      • Do this instead:
        <p>
        <input type="submit" value="log-in">
        </p>
  • Attribute names need to be lowercase: Don't write the name of the attribute upercasse, otherwise you might have problem.
    • Example:
      • Don't do this:
        <p>
        <input TYPE="submit" VALUE="log-in">
        </p>
      • Do this instead:
        <p>
        <input type="submit" value="log-in">
        </p>
  • Element types have private attribute: Each element should have their own attributes according to the types.
    • Example:
      • Don't do this:
        <p>
        <input type="submit,log-in">
        </p>
      • Do this instead:
        <p>
        <input type="submit" value="log-in">
        </p>
  • Public attributes can go on any elements like: class; id=hello
  • No double quotes inside of doubles quotes: Don't use double quotes to wrap a double quotes.
    • Example:
      • Don't do this:
        <p>
        <input type=""submit"" value=""log-in"">
        </p>
      • Do this instead:
        <p>
        <input type="submit" value="log-in">
        </p>
