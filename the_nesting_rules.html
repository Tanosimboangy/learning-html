List of the nesting rules

These are some of the nesting rules:

  • All tags must be closed: To close a tag, repeat the opening tag exactly, but insert a slash in front of the tag name.
    • Example:
      • Don't do this:
        <div> Ernest Hemingway
        <p> Roses are Red
      • Do this instead:
        <div> Ernest Hemingway </div>
        <p>Roses are Red </p>
  • All tags must pe properly nested: When you nest one tag within another, pay attention to the order in which you open each tag, and then close the tags in the reverse order. If you open element A and then element B, you must first close B before closing A.
    • Example:
      • Don't do this:
        <div><p>Put</div></p>
        <section><h1>Text</section></h1>
      • Do this instead:
        <div><p>Put</p></div>
        <section><h1>Text</h1></section>
  • Tag names have strict limits: Tag names can’t start with a number, or punctuation, except for the underscore character (_).
    • Example:
      • Don't do this:
        <01_article>
        <"article">
      • Do this instead:
        <article>
        <_article>
  • Tag names are case sensitive: Uppercase and lowercase matter in HTML. Opening and closing tags must match exactly. For example, <root> is the opening tag the closing tag should be </root>.
    • Example:
      • Don't do this:
        <label>Hemingway</LABEL>
        <Label>Hemingway</lABEL>
      • Do this instead:
        <label>Hemingway</label>
        <LABEL>Hemingway</LABEL>
  • Attribute values must appear within quotes: Attribute values modify a tag or help identify the type of information being tagged.
    • Example:
      • <chapter number="1">
        <artist title="author" nationality="USA">
  • Avoid HTML Tags (Optional): Because you can name tags anything you want, you could use tags reserved for HTML markup, such as <h1>, <p>, <li>.
